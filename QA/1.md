1. Basic CUDA Concepts
  
- What is the difference between a thread, block, and grid? (Threads are individual execution units, blocks are groups of threads that can communicate/synchronize, and grids are collections of blocks).
  
- What is a warp? (The basic unit of execution, usually 32 threads, which run in SIMT—Single Instruction, Multiple Thread—mode).
  
- How do you synchronize threads? (Using __syncthreads() for threads within a block, or CUDA events/streams for communication across blocks).
  
- What is a CUDA kernel? (A function executed on the GPU, designed to run in parallel). 

 
2. CUDA Memory Architecture
  
- What are the different types of memory in a GPU? (Global, Shared, Local, Register, Constant, Texture).
  
- What is memory coalescing? (When threads in a warp access consecutive global memory locations, reducing memory requests. Crucial for performance).

- What is the difference between Shared Memory and Registers? (Shared memory is accessible by all threads within a block; registers are private to each thread and faster).

- When would you use __shared__ memory? (To speed up access when data is reused by threads within the same block). 

  
3. Optimization and Performance
  
- What is Kernel Occupancy? (The ratio of active warps to the maximum number of supported warps, influencing latency hiding).
  
- How do you optimize a CUDA kernel? (By maximizing memory coalescing, utilizing shared memory, avoiding bank conflicts, reducing divergence, and maximizing occupancy).
  
- What is warp divergence? (When threads within a warp take different branches of code, forcing serial execution of branches).
  
- What are CUDA streams? (A sequence of operations that execute in order, allowing for concurrency between data transfer and kernel execution).
